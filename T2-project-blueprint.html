02

Daily Operating Workflow

08:00 – 09:29 ET · PRE-MARKET
Data Fetch + Regime Assessment
Pull overnight NQ data, VIX level, pre-market high/low/trend. Classify VIX bucket (low/normal/elevated/extreme). Set preliminary bias. No trades.
09:30 – 10:00 ET · OPEN WATCH
First 30-Min Candle Formation
Monitor and record first 30-min candle. Green → LONG bias (73% historical). Red → SHORT bias (67% historical). Lock session direction. Mark ORB high/low.
10:00 – 11:30 ET · PRIME WINDOW
Active Trading — Setup Scan
Strategy engine scans for qualifying setups. Confluence check: bias alignment + indicator stack. Risk gate check before every order. Max 3 trades this window.
11:30 – 13:30 ET · CHOP ZONE
Reduced Activity / Monitor Only
Lunch hour. Volatility typically drops. System reduces position size or pauses. Only takes A+ setups. 1 trade max.
13:30 – 15:00 ET · AFTERNOON
Power Hour Reactivation
Volume picks up. System reactivates. VWAP reclaim / reject setups. Max 2 trades. Target hit or daily stop hit → system shuts down for day.
15:45 ET · HARD CLOSE
Force Flat — No Overnight Holds
All positions closed. No exceptions. System logs final P&L.
16:00 ET · EOD
EOD Report → Telegram
P&L, win rate, trade log, drawdown status, next day prep notes pushed to @Benkboxerbot.
TRADE DECISION LOGIC — PSEUDO-FLOW
ON setup_signal_detected:
// GATE 1: Regime
IF signal_direction != session_bias:
SKIP → log "contra-bias, no trade"
END
 
// GATE 2: Confluence
IF confluence_score < threshold:
SKIP → log "low confluence"
END
 
// GATE 3: Risk
IF daily_pnl <= -$100:
KILL SWITCH → halt all trading
IF daily_pnl >= +$100:
TARGET HIT → halt, log win day
IF trade_count >= max_daily_trades:
SKIP → log "max trades reached"
 
// EXECUTE
CALCULATE position_size(account, stop_pts)
SEND limit_order(entry, stop, target)
MONITOR fill → update P&L
ON close: log trade to SQLite

04

Risk Rules — Non-Negotiable

Daily Loss Limit
System halts all activity at -$100 net for the day. No override. No "one more trade." Hard stop.
Daily Target Lock
System halts at +$100 net. Bank the win. No greed extension. Consistency beats home runs.
Max Trades Per Day
Hard cap at 6 trades per session. Prevents revenge trading and overexposure in chop.
No Overnight Holds
All positions flat by 15:45 ET. Micro futures have overnight gap risk. System is intraday only.
Contra-Bias Trades
No trades against session bias unless confluence score is exceptional (>85). Bias is the filter, not a suggestion.
Chop Detection
If VWAP cross count exceeds 4 in 30 minutes, system flags chop mode and reduces to 1-contract max.

07

Known Blockers

⚠ BROKER API DECISION NOT MADE
Tradovate vs Schwab must be decided before P2. Tradovate has a native WebSocket and is purpose-built for futures automation. Schwab is already in T2 context but its futures API is less mature. This is the single biggest upstream decision — everything else builds on it.
⚠ NO HISTORICAL MNQ DATA IN SQLITE YET
P1 backtest requires clean 1-min and 2-min OHLCV bars for MNQ. Source: Tradovate historical download, or free alternative via Databento free tier or Polygon.io. Must be resolved before any backtest runs.
⚠ BACKTEST OVERFITTING RISK
Any strategy that shows great numbers on 2020–2021 data will likely fail live. Validation must include regime-separated testing: trending year vs. choppy year vs. crash period. Debate agents must be explicitly prompted to argue against curve-fitting.
⚠ COMMISSION + SLIPPAGE KILLS THIN EDGES
$2.50 round trip per MNQ contract. Any strategy showing <5 points expected value per trade is marginal at best after realistic fills. All backtests must model at least 0.5 point of slippage per fill plus commissions.
